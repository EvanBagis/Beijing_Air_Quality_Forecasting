## Introduction
This repository presents the implementation of two distinct forecasting frameworks applied to the [Beijing Air Quality Dataset](https://archive.ics.uci.edu/dataset/501/beijing+multi+site+air+quality+data). These frameworks leverage different methodologies to address the challenge of predicting air quality parameters over time.

- **Approach 1: Incremental recursive forecasting** uses the [river](https://riverml.xyz/0.21.0/) python library to implement an incremental forecasting model tailored to predict air quality parameters 12 hours into the future. The model adopts a recursive forecasting strategy where it initially assimilates the latest 24 observations from the time series to forecast the subsequent value. Subsequently, the predicted value becomes the most recent observation for predicting the value two steps ahead, and this process continues iteratively.

- **Approach 2: Batch multioutput forecasting**. In contrast, the second approach utilizes models from the [sklearn](https://scikit-learn.org/stable/)library to conduct multioutput forecasting. This framework simultaneously forecasts future values for multiple time series, each corresponding to a specific air quality monitoring station. By considering lagged features from each time series, the model predicts future hours for all time series concurrently. An advantage of this approach lies in its ability to leverage correlations between different time series, enhancing the forecasting accuracy.

- **Machine learning model**. The incremental recursive approach integrates a random forest regressor augmented by a k-nearest neighbors model with k=9 as its foundational framework. Conversely, the batch multioutput methodology adopts the conventional random forest algorithm comprising 100 regression trees. Furthermore, the latter approach holds the potential for refinement through the incorporation of ensemble learning techniques, such as stacking, which involve the integration of multiple machine learning algorithms to augment predictive performance.

- **Evaluation**:Both approaches undergo evaluation using various performance metrics, including the root mean squared error (RMSE), the mean absolute error (MAE), the coefficient of determination (r2), the Pearson and Spearman correlation coefficients, the mean bias error (MBE) and the index of aggreement (IA).

- **Plots & tables**: Plots are generated for each forecasting horizon, providing insights into the model's performance over time. For instance, a forecast for hour = 3 corresponds to predictions made three hours after the final observation in the input vectors. Additionally, tables containing concatenated predictions over the entire time series interval showcase the model's overall predictive capabilities.

### Conceptual framework

- **Approach 1: Incremental recursive forecasting**. This framework follows a sequential process where the model initially utilizes the first N lag values from the time series to formulate the input vector ( e.g. y(t-24) ... y(t) ). The model then forecasts the target value for the subsequent hour. Subsequently, it incorporates the N-1 lag values and the previous prediction ( e.g. y(t-23) ... y(t+1) ) to forecast the target value for the following hour. This iterative process continues for the selected forecasting horizon (e.g., 12 hours ahead). After forecasting all values within the horizon, the model transitions into the learning mode, utilizing the same input vectors for training. This incremental learning process repeats until the entire time series is processed.

![](https://github.com/EvanBagis/Beijing_Air_Quality_Forecasting/blob/master/imgs/incremental_recursive_forecasting.png)

- **Approach 2: Batch multioutput forecasting**. In this framework, the input vector comprises N lag values from each of the multiple time series. Similarly, the forecasts encompass the next M hours' values for all time series. Both input and output vectors are flattened, and the dataset is split into train and test sets. Consequently, the input and output vector shapes are standardized, ensuring compatibility across the dataset.

![](https://github.com/EvanBagis/Beijing_Air_Quality_Forecasting/blob/master/imgs/batch_multioutput_forecasting.png)

### Conclusions

The comprehensive utilization of data from various monitoring stations empowers the multioutput framework to leverage the collective information inherent in diverse geographical locations. This enables the model to capture a more comprehensive understanding of the spatiotemporal dynamics influencing AQ levels. In contrast, the recursive approach, by virtue of its sequential nature, is limited in its ability to incorporate information from multiple sources simultaneously. Consequently, the multioutput approach demonstrates superior performance in forecasting AQ levels across diverse spatial domains.

Moreover, the iterative nature of the recursive approach predisposes it to compounding errors over successive time steps. As past predictions are recursively utilized to inform future forecasts, any inaccuracies or deviations from the actual values propagate throughout the forecasting horizon, leading to a notable degradation in predictive accuracy. In contrast, the multioutput framework circumvents this issue by concurrently forecasting AQ levels for multiple stations, thereby mitigating the impact of error accumulation and ensuring more robust predictions across the entire forecasting horizon.

In essence, the superiority of the multioutput approach underscores the significance of harnessing information from multiple monitoring stations to enhance the predictive accuracy of AQ forecasting models. By leveraging the collective insights gleaned from diverse spatial sources, the multioutput framework offers a more holistic and accurate portrayal of atmospheric AQ dynamics, thereby facilitating informed decision-making and proactive mitigation strategies to safeguard public health and environmental quality.
